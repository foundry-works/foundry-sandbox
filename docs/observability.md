# Observability Guide

This guide covers metrics, logging, and alerting for the git safety layer.

## Prometheus Metrics

The foundry-git-safety server exposes metrics at its health endpoint.

### Git Operation Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `git_safety_requests_total` | Counter | sandbox_id, action, decision | Total git operations processed |
| `git_safety_request_duration_seconds` | Histogram | sandbox_id, action | Operation latency in seconds |

**Latency Buckets**: 10ms, 50ms, 100ms, 250ms, 500ms, 1s, 2.5s, 5s, 10s

### Rate Limiter Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `git_safety_rate_limit_remaining` | Gauge | sandbox_id | Remaining rate limit tokens |

### Policy Enforcement Metrics

| Metric | Type | Labels | Description |
|--------|------|--------|-------------|
| `git_safety_policy_results_total` | Counter | sandbox_id, policy, decision | Policy enforcement decisions |

**Policy Labels**: `command_allowlist`, `branch_isolation`, `protected_branch`, `file_restriction`, `rate_limit`

## Structured Logging

The foundry-git-safety server uses JSON-formatted logs for machine parsing.

### Log Format

```json
{
  "timestamp": "2026-04-19T13:45:00.123456Z",
  "level": "INFO",
  "message": "Git operation allowed",
  "logger": "git_safety.server",
  "request_id": "req-abc123",
  "sandbox_id": "repo-feature-branch",
  "action": "push",
  "decision": "allowed"
}
```

### Log Fields

| Field | Description |
|-------|-------------|
| `timestamp` | ISO 8601 timestamp with microseconds |
| `level` | Log level (DEBUG, INFO, WARNING, ERROR) |
| `message` | Human-readable log message |
| `logger` | Logger name (usually module path) |
| `request_id` | Unique request identifier for tracing |
| `sandbox_id` | Source sandbox identifier |
| `action` | Git subcommand (push, fetch, checkout, etc.) |
| `decision` | Outcome: `allowed`, `denied`, or `error` |
| Additional fields | Context-specific data |

### Configuration

| Environment Variable | Default | Description |
|---------------------|---------|-------------|
| `GIT_SAFETY_LOG_LEVEL` | INFO | Minimum log level |
| `GIT_SAFETY_LOG_FORMAT` | json | Output format (json or text) |

### Correlation IDs

Each request is assigned a `request_id` for end-to-end tracing. The ID is generated by the git wrapper and propagated through the HMAC-signed request headers.

## Recommended Alerts

> These alerts are relevant for team or multi-tenant deployments where Prometheus is configured. For single-user setups, the [troubleshooting procedures](operations.md#troubleshooting) are typically sufficient.

| Alert | Expression | Severity | Trigger |
|-------|-----------|----------|---------|
| Git safety server down | `up{job="git-safety"} == 0` | Critical | Server unreachable for 1m |
| High denial rate | `rate(git_safety_policy_results_total{decision="denied"}[5m]) > 10` | Warning | >10 denials per second for 5m |
| Rate limit low | `git_safety_rate_limit_remaining < 10` | Warning | Few tokens remaining for 2m |
| High latency | `p95 git operation duration > 2s` | Warning | Sustained for 5m |
| HMAC auth failures | `rate(git_safety_requests_total{decision="error"}[5m]) > 5` | Critical | Auth errors sustained for 5m |
| Wrapper tamper detected | any `wrapper_tamper` event in decision log | Critical | Tamper event within 5 min |

For debugging procedures, see [Operations: Troubleshooting](operations.md#troubleshooting).

## Metrics Endpoint

Access metrics from the git safety server:

```bash
# Health/metrics check
curl http://127.0.0.1:8083/health

# From a sandbox (via sbx proxy)
sbx exec <name> -- curl --proxy http://gateway.docker.internal:3128 http://host.docker.internal:8083/health
```

## Git Operation Audit Logs

The git safety server emits structured audit log entries for all git operations. These logs include detailed context for security investigation.

### Audit Log Fields

| Field | Description |
|-------|-------------|
| `event` | Event type (e.g., `git_operation`) |
| `action` | Git subcommand executed (e.g., `push`, `fetch`, `checkout`) |
| `decision` | Outcome: `allowed`, `denied`, or `error` |
| `sandbox_id` | Sandbox identity from HMAC authentication |
| `reason` | Explanation for denied operations (e.g., branch isolation violation) |
| `matched_rule` | Which policy rule matched (e.g., `protected_branch`, `branch_isolation`) |
| `command_args` | Git command arguments (sanitized) |
| `exit_code` | Process exit code for executed commands |
| `request_id` | Unique request identifier for tracing |

### Example Queries

```bash
# Find all denied git operations
journalctl -u foundry-git-safety | grep '"decision": "denied"'

# Find branch isolation violations
journalctl -u foundry-git-safety | grep 'branch_isolation'

# Find operations by a specific sandbox
journalctl -u foundry-git-safety | grep '"sandbox_id": "repo-feature-branch"'

# Find protected branch enforcement events
journalctl -u foundry-git-safety | grep 'protected_branch'
```

### Wrapper Audit Logs

The git wrapper script inside sandboxes logs to stderr. View via:

```bash
sbx exec <name> -- bash -c 'git --version 2>&1'
```

## Wrapper Tamper Events

The watchdog emits a structured event to the decision log on every wrapper checksum mismatch:

```json
{
  "timestamp": "2026-04-20T12:34:56.789000+00:00",
  "sandbox": "repo-feature-branch",
  "rule": "wrapper_integrity",
  "verb": "wrapper_tamper",
  "outcome": "reinjected",
  "expected_sha256": "abc123...",
  "actual_sha256": "def456..."
}
```

### Event Fields

| Field | Description |
|-------|-------------|
| `timestamp` | ISO 8601 UTC timestamp |
| `sandbox` | Sandbox name |
| `rule` | Always `wrapper_integrity` |
| `verb` | Always `wrapper_tamper` |
| `outcome` | `reinjected` (success) or `reinject_failed` (failure) |
| `expected_sha256` | Expected wrapper checksum |
| `actual_sha256` | Checksum found in the sandbox |

### Querying

```bash
# Recent tamper events from the decision log
cast diagnose

# Filter the JSONL log directly
grep 'wrapper_tamper' ~/.foundry/logs/decisions.jsonl | jq .
```
